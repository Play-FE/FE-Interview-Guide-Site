基本数据类型 ​

JavaScript 有 7 种基本数据类型，分别是：

  1. undefined（未定义） 表示变量未赋值时的默认值。

  2. null（空值） 表示变量的值为空或不存在。

  3. boolean（布尔值） 表示逻辑上的真或假。可以是 true 或 false。

  4. number（数字） 表示整数或浮点数。JavaScript 中的所有数字都是 64 位浮点数。

  5. string（字符串） 表示文本数据。可以使用单引号（'）或双引号（"）来定义字符串。

  6. symbol（符号） ES6（ECMAScript 2015）中的新类型，表示独一无二的值。

  7. bigint（大整数） ES2020 中的新类型，用于表示任意精度的整数。

数据类型 ​

JavaScript 中有两类数据类型：基本数据类型（Primitive Types）和对象类型（Object Types）。

基本数据类型（Primitive Types） ​

基本数据类型是不可变的，表示简单的值。在 JavaScript 中，有 7 种基本数据类型：

  • undefined 表示变量未赋值的默认值。
  • null 表示变量的值为空或不存在。
  • boolean 表示逻辑上的真或假。可以是 true 或 false。
  • number 表示整数或浮点数。JavaScript 中的所有数字都是 64 位浮点数。
  • string 表示文本数据。可以使用单引号（'）或双引号（"）来定义字符串。
  • symbol ES6（ECMAScript 2015）中的新类型，表示独一无二的值。
  • bigint ES2020 中的新类型，用于表示任意精度的整数。

对象类型（Object Types） ​

对象类型是可变的，用于存储和组织复杂的数据和功能。JavaScript 中的对象是键值对的集合，可以包含属性和方法。

  • Object 通用的对象类型，可以包含属性和方法。
  • Array 表示有序的集合，每个元素可以通过索引访问。
  • Function 表示可调用的代码块，可以接受参数并返回值。
  • Date 表示日期和时间。
  • RegExp 表示正则表达式，用于字符串匹配。
  • 其他内置对象，如 Map、Set、Promise 等，提供了各种功能和数据结构。

JavaScript 是一种动态类型语言，变量在运行时可以被赋予不同的数据类型。此外，JavaScript 还具有一些特殊的值，如 Infinity、NaN 等，用于表示特殊的数值情况。

基本数据类型和对象类型的区别 ​

JavaScript 中的两类数据类型分别是基本数据类型（Primitive Types）和对象类型（Object Types）。它们在多个方面有着不同的特点，以下是它们之间的主要区别：

  • 存储方式

    • 基本数据类型：存储的是简单的数据值，是不可变的。每个变量直接包含其值，而不是引用其他对象。
    • 对象类型：存储的是引用类型的值，是可变的。变量存储的是对实际对象的引用，而不是直接的数据值。

  • 可变性

    • 基本数据类型：不可变，一旦创建，其值无法被修改。
    • 对象类型：可变，可以添加、修改、删除属性，以及改变对象的状态。

  • 比较方式

    • 基本数据类型：通过值比较。两个基本数据类型的变量只要值相等，它们就被认为是相等的。
    • 对象类型：通过引用比较。两个对象只有在它们引用同一个实际对象时才被认为是相等的。

  • 传递方式

    • 基本数据类型：通过值传递。函数参数传递基本数据类型时，是将实际的值复制给参数。
    • 对象类型：通过引用传递。函数参数传递对象时，是将引用复制给参数，因此函数内部可以修改对象的属性。

  • 内存分配

    • 基本数据类型：存储在栈内存中，直接按值访问，效率较高。
    • 对象类型：存储在堆内存中，需要通过引用访问，因此可能涉及到更多的内存管理工作。

  • 复制方式

    • 基本数据类型：复制的是值，互不影响。
    • 对象类型：复制的是引用，复制后的变量与原变量共享同一个对象。

堆（Heap）和栈（Stack） ​

在 JavaScript 中，堆（Heap）和栈（Stack）是两种主要的内存分配区域，用于存储不同类型的数据和执行不同的任务。

堆（Heap） ​

  • 特点：

    • 堆是用于存储动态分配的数据的区域。
    • 通过引用访问堆中的数据，而不是直接按值访问。
    • 存储对象、数组等复杂的数据结构，以及通过 new 关键字创建的对象。
    • 内存分配和释放由开发人员手动管理或由垃圾回收器（Garbage Collector）负责。

  • 使用场景：

    • 存储动态分配的数据，如对象、数组等。
    • 对象的生命周期通常比栈中的数据更长。

栈（Stack） ​

  • 特点：

    • 栈是用于存储执行上下文和基本数据类型的区域。
    • 存储基本数据类型（如数字、字符串、布尔值）和引用类型的引用。
    • 通过栈帧（Stack Frame）来跟踪函数调用和执行上下文的状态。
    • 内存的分配和释放是自动进行的，由 JavaScript 引擎负责。

  • 使用场景：

    • 存储基本数据类型和引用类型的引用。
    • 跟踪函数的调用和执行上下文，确保函数的正确执行。

let 和 const 是 ES6（ECMAScript 2015）引入的两个用于声明变量的关键字，相较于传统的 var，它们在作用域、变量重新赋值和暂时性死区等方面有一些不同之处。

总结 ​

  • 使用 let 声明的变量可以重新赋值，而 const 声明的变量是常量，不可重新赋值。
  • 块级作用域是指变量在块或语句中有效，而不是函数或全局作用域。

null 和 undefined ​

  • undefined 代表的含义是未定义，null 代表的含义是空对象。一般变量声明了但还没有定义的时候会返回 undefined，null 主要用于赋值给一些可能会返回对象的变量，作为初始化。

  • undefined 在 JavaScript 中不是一个保留字，这意味着我们可以使用 undefined 来作为一个变量名，这样的做法是非常危险的，它会影响我们对 undefined 值的判断。但是我们可以通过一些方法获得安全的 undefined 值，比如说 void 0。

  • 当我们对两种类型使用 typeof 进行判断的时候，Null 类型化会返回 “object”，这是一个历史遗留的问题。当我们使用双等号对两种类型的值进行比较时会返回 true，使用三个等号时会返回 false。

typeof NaN ​

  • NaN 意指“不是一个数字”（not a number），NaN 是一个“警戒值”（sentinel value，有特殊用途的常规值），用于指出数字类型中的错误情况，即“执行数学运算没有成功，这是失败后返回的结果”。

其他值到字符串的转换规则 ​

  • Null 和 Undefined 类型，null 转换为 "null"，undefined 转换为 "undefined"。

  • Boolean 类型，true 转换为 "true"，false 转换为 "false"。

  • Number 类型的值直接转换，不过那些极小和极大的数字会使用指数形式。

  • Symbol 类型的值直接转换，但是只允许显式强制类型转换，使用隐式强制类型转换会产生错误。

对普通对象来说，除非自行定义 toString() 方法，否则会调用 Object.prototype.toString() 来返回内部属性 [[Class]] 的值，如 "[object Object]"。如果对象有自己的 toString() 方法，字符串化时就会调用该方法并使用其返回值。

其他值到数字值的转换规则 ​

  • Undefined 类型的值转换为 NaN。

  • Null 类型的值转换为 0。

  • Boolean 类型的值，true 转换为 1，false 转换为 0。

  • String 类型的值转换如同使用 Number() 函数进行转换，如果包含非数字值则转换为 NaN，空字符串为 0。

  • Symbol 类型的值不能转换为数字，会报错。

  • 对象（包括数组）会首先被转换为相应的基本类型值，如果返回的是非数字的基本类型值，则再遵循以上规则将其强制转换为数字。

为了将值转换为相应的基本类型值，抽象操作 ToPrimitive 会首先（通过内部操作 DefaultValue）检查该值是否有 valueOf() 方法。如果有并且返回基本类型值，就使用该值进行强制类型转换。如果没有就使用 toString() 的返回值（如果存在）来进行强制类型转换。

如果 valueOf() 和 toString() 均不返回基本类型值，会产 TypeError 错误。

// 2、随机从原数组抽取一个元素，加入到新数组。
function randomSort(arr) {
  var result = [];

  while (arr.length > 0) {
    var randomIndex = Math.floor(Math.random() * arr.length);
    result.push(arr[randomIndex]);
    arr.splice(randomIndex, 1);
  }

  return result;
}

// 3、随机交换数组内的元素（洗牌算法类似）
function randomSort(arr) {
  var index,
    randomIndex,
    temp,
    len = arr.length;

  for (index = 0; index < len; index++) {
    randomIndex = Math.floor(Math.random() * (len - index)) + index;
    temp = arr[index];
    arr[index] = arr[randomIndex];
    arr[randomIndex] = temp;
  }

  return arr;
}

// ES6
function randomSort(array) {
  let length = array.length;

  if (!Array.isArray(array) || length <= 1) return;

  for (let index = 0; index < length - 1; index++) {
    let randomIndex = Math.floor(Math.random() * (length - index)) + index;

    [array[index], array[randomIndex]] = [array[randomIndex], array[index]];
  }

  return array;
}

作用域链 ​

JavaScript 的作用域链（Scope Chain）是指在代码中查找变量和函数的过程中所形成的嵌套作用域的链式结构。作用域链的形成是由代码中函数和块级作用域的嵌套关系所决定的。

作用域链的形成过程 ​

  • 当执行一个函数时，JavaScript 引擎会创建一个包含该函数参数、局部变量和内部函数引用的对象，称为活动对象（Activation Object），也被称为变量对象（Variable Object）。
  • 作用域链就是由当前执行上下文的活动对象和其父级执行上下文的活动对象构成的链式结构。
  • 作用域链的最前端始终是当前执行上下文的活动对象，最末端是全局执行上下文的活动对象。

作用域链的查找规则 ​

  • 当需要查找一个变量或函数时，JavaScript 引擎首先在当前执行上下文的活动对象中查找。
  • 如果找不到，就沿着作用域链一级一级向上查找，直到找到或者到达全局作用域。
  • 如果在全局作用域中仍然找不到，则返回 ReferenceError。

闭包和作用域链 ​

  • 闭包是指一个函数可以访问其外部函数的变量，即使在外部函数执行完毕后仍然保持对这些变量的引用。
  • 闭包的存在导致了作用域链的延长，即在闭包内部可以访问外部函数的变量，形成一个更深层次的作用域链。

  • 操作：DOM 提供了一系列的 API，通过这些 API 可以增删改查文档的元素、属性和文本。

事件 ​

在 JavaScript 中，事件是指在程序执行过程中发生的一些事情，例如用户点击按钮、页面加载完成等。事件可以触发相应的事件处理函数，以执行特定的操作。以下是一些常见的 JavaScript 事件：

  1. 鼠标事件：
  • click: 当用户点击元素时触发。
  • mouseover和mouseout: 当鼠标移入或移出元素时触发。
  • mousedown、mouseup: 当鼠标按下或释放时触发。

  • keydown和keyup: 当用户按下或释放键盘上的键时触发。

  • submit: 当用户提交表单时触发。
  • change: 当表单元素的值发生变化时触发。
  • input: 当输入框中的值发生变化时触发。
  1. 文档/窗口事件：
  • load: 当文档或页面完成加载时触发。
  • unload: 当用户离开页面时触发。
  • resize: 当窗口大小发生变化时触发。
  1. 焦点事件：
  • focus和blur: 当元素获得或失去焦点时触发。
  1. 事件处理方式：
  • 内联事件处理： 直接在 HTML 标签中通过on属性定义，如<button onclick="myFunction()">Click me</button>。

  1. 事件对象：
  • 当事件发生时，会创建一个事件对象，其中包含有关事件的信息。事件对象作为参数传递给事件处理函数，可以使用该对象获取有关事件的详细信息，如触发事件的元素、鼠标位置等。

事件委托 ​

事件委托（Event Delegation）是一种利用事件冒泡的特性，将事件处理程序绑定到一个父元素，从而减少事件处理程序的数量的技术。通过这种方式，可以在单一的事件处理程序上管理多个子元素的事件，提高性能并简化代码。

工作原理 ​

  1. 事件冒泡： 在 DOM 中，当一个事件（如点击）发生在某个元素上时，事件会从该元素开始向上冒泡至 DOM 树的根部。这是浏览器处理事件的默认行为。

  2. 事件委托： 将事件处理程序绑定到父元素，然后利用事件冒泡的过程来捕获子元素上的事件。

示例 ​

考虑一个 ul 元素包含多个 li 元素的列表，我们想要为每个 li 元素添加点击事件处理程序。使用事件委托的方式如下：

  document.getElementById('parentList').addEventListener('click', function(event) {
    // 检查点击的是否是 li 元素
    if (event.target.tagName === 'LI') {

    if (proto === prototype) return true;

浏览器的缓存机制 ​

浏览器的缓存机制是为了提高性能和降低网络负载而设计的。它通过保存已经获取的资源，以便将来再次使用，减少对服务器的请求次数，从而提升网页加载速度。浏览器的缓存机制主要包括两类：HTTP 缓存和浏览器存储。

HTTP 缓存机制 ​

HTTP 缓存是通过 HTTP 头部来控制的，主要包括以下几个头部字段：

  • Expires： 指定资源的到期时间，即过了这个时间后，浏览器会向服务器发起请求重新获取资源。

  • Cache-Control： 提供了更灵活的控制，包括 public、private、max-age 等，用于指定缓存是否可以由所有人使用，是否可以存储，以及缓存的最大有效时间。

  • Last-Modified 和 If-Modified-Since： 服务器通过 Last-Modified 头部告诉浏览器资源的最后修改时间，浏览器通过 If-Modified-Since 头部将最后获取资源的时间发送给服务器，服务器比较这两个时间，如果没有变化，则返回 304 Not Modified 状态码，告诉浏览器直接使用缓存。

  • ETag 和 If-None-Match： 服务器通过 ETag 头部生成一个唯一的标识符，浏览器通过 If-None-Match 头部将上次获取资源时获得的 ETag 发送给服务器，如果没有变化，则返回 304 Not Modified 状态码。

缓存优先级 ​

浏览器的缓存机制通常遵循一定的优先级顺序：

  1. 强缓存（Cache-Control、Expires）： 浏览器首先检查强缓存，如果资源没有过期，直接从本地缓存中获取，不发起请求。

  2. 协商缓存（Last-Modified、If-Modified-Since、ETag、If-None-Match）： 如果资源过期或没有强缓存，浏览器发起请求，服务器通过协商缓存验证资源是否有变化，如果没有变化，返回 304 状态码，浏览器使用缓存。

  3. 本地存储和会话存储： 如果以上两者都未命中，浏览器尝试从本地存储或会话存储中获取数据，如果存在，则使用。

浏览器的同源政策 ​

同源政策（Same-Origin Policy）是一种浏览器安全机制，用于限制一个网页文档或脚本如何能够与其他来源的资源进行交互。同源指的是协议、域名和端口号都相同，如果不满足这三者之一，就被认为是跨域请求，同源政策将对其进行限制。同源政策的目的是防止恶意网站通过恶意代码访问用户的敏感信息或进行恶意操作。

同源政策主要限制以下行为：

  • Cookie、LocalStorage 和 IndexDB 的访问： 不同源的网页无法访问彼此的 Cookie、LocalStorage 和 IndexDB 数据。

  • DOM 的限制： 不同源的网页无法通过脚本直接访问对方的 DOM。

  • Ajax 请求的限制： 使用 XMLHttpRequest 发起的 Ajax 请求受到同源政策的限制，只能向同源服务器发送请求。

以下是关于 V8 引擎的垃圾回收机制的主要特点：

分代垃圾回收 ​

V8 引擎采用分代垃圾回收策略，将内存分为新生代（Young Generation）和老生代（Old Generation）两个代。新生代中的对象生命周期较短，老生代中的对象生命周期较长。

  • 新生代： 使用副垃圾回收器（Scavenge Collector），采用复制算法。将新生代内存分为两个区域：From 空间和 To 空间。对象首先被分配到 From 空间，当 From 空间满时，会启动垃圾回收，将存活的对象复制到 To 空间，然后清空 From 空间。在复制的同时，对象的存活时间增长，最终会晋升到老生代。

  • 老生代： 使用主垃圾回收器（Mark-Sweep and Mark-Compact Collector）。主要分为标记阶段、清除阶段和整理阶段。标记阶段用于标记存活的对象，清除阶段用于清除未标记的对象，整理阶段用于压缩内存空间，减少内存碎片。

增量标记 ​

V8 引擎采用增量标记（Incremental Marking）的方式执行垃圾回收，将整个垃圾回收过程分解为多个小步骤，穿插在 JavaScript 执行中。这样可以减小单次垃圾回收的时间，降低对用户代码的影响。

空闲时执行垃圾回收 ​

V8 引擎利用 JavaScript 执行空闲时间来执行垃圾回收。当浏览器空闲时，例如用户停止滚动或打开其他标签页，V8 将尽可能地执行垃圾回收操作，以最小化对用户交互的干扰。

快速分配 ​

为了提高对象分配的效率，V8 引擎采用了快速分配的策略。通过将新对象分配在新生代的 To 空间，而不是等待垃圾回收的完成，从而避免了等待新生代的垃圾回收操作。

内存限制 ​

为了防止内存过度膨胀，V8 引擎会设置内存限制。当内存占用接近限制时，V8 会主动触发垃圾回收，尽可能释放不再使用的内存。

前端路由 ​

  • 前端路由就是把不同路由对应不同的内容或页面的任务交给前端来做，之前是通过服务端根据 url 的不同返回不同的页面实现的。
  • 在单页面应用，大部分页面结构不变，只改变部分内容的使用。
  • 前端路由一共有两种实现方式，一种是通过 hash 的方式，一种是通过使用 pushState 的方式。
  • 优点：用户体验好，不需要每次都从服务器全部获取，快速展现给用户。
  • 缺点：单页面无法记住之前滚动的位置，无法在前进，后退的时候记住滚动的位置。

Polyfill 的目标是在尽可能少的代码和修改的情况下，提供对新特性的支持。它可以是一个 JavaScript 库、脚本或代码片段，通常由开发者或社区提供。

节流 ​

定义：规定一个单位时间，在这个单位时间内，只能有一次触发事件的回调函数执行，如果在同一个单位时间内某事件被触发多次，只有一次能生效。

使用场景：窗口 resize、scroll、输入框 input、频繁点击等

js
function throttle(fn, delay) {
  var preTime = Date.now();

  return function () {
    var context = this,
      args = arguments,
      nowTime = Date.now();

    // 如果两次时间间隔超过了指定时间，则执行函数。
    if (nowTime - preTime >= delay) {
      preTime = Date.now();
      return fn.apply(context, args);
    }
  };
}

防抖 ​

定义：在事件被触发 n 秒后再执行回调，如果在这 n 秒内事件又被触发，则重新计时。

function debounce(fn, wait) {
  var timer = null;

  return function () {
    var context = this,
      args = arguments;

    // 如果此时存在定时器的话，则取消之前的定时器重新记时
    if (timer) {
      clearTimeout(timer);
      timer = null;
    }

    // 设置定时器，使事件间隔指定事件后执行
    timer = setTimeout(() => {
      fn.apply(context, args);
    }, wait);
  };
}

事件循环 ​

JavaScript 的事件循环（Event Loop）是一种异步编程模型，用于处理事件和回调函数。它确保在单线程执行环境中能够处理异步任务，保持响应性并避免阻塞主线程。事件循环的执行过程主要涉及主线程、回调队列、任务队列、微任务队列等组件。

调用栈（Call Stack） ​

调用栈用于追踪代码执行的上下文，它保存了当前执行上下文的信息。每当一个函数被调用，它的执行上下文被推入调用栈，函数执行完成后，对应的执行上下文被弹出。

任务队列（Task Queue） ​

任务队列用于存储异步任务的回调函数。每个回调函数都被包装成一个任务，并添加到任务队列中。任务队列分为宏任务队列和微任务队列。

  • 宏任务队列（Task Queue）： 存储宏任务（macrotasks），例如定时器回调、事件回调等。

  • 微任务队列（Microtask Queue）： 存储微任务（microtasks），例如 Promise 的回调、MutationObserver 回调等。微任务在主线程执行栈为空时立即执行，优先于宏任务。

事件循环的阶段 ​

JavaScript 的事件循环被分为不同的阶段，每个阶段都有特定的任务和执行顺序：

  1. 同步任务执行阶段（Synchronous Execution）： 从上到下逐行执行主线程中的同步任务。

  2. 宏任务执行阶段（Macro Task Execution）： 依次执行宏任务队列中的一个宏任务。

  3. 微任务执行阶段（Micro Task Execution）： 依次执行微任务队列中的所有微任务。

  4. 渲染阶段（仅浏览器环境）： 如果有与 UI 渲染相关的任务，会执行这一阶段，例如页面重绘。

事件循环的执行过程 ​

  1. 执行同步任务： 从调用栈中执行同步任务，直至调用栈为空。

  2. 执行微任务： 如果存在微任务，依次执行微任务队列中的所有微任务。

  3. 执行宏任务： 从宏任务队列中取出一个宏任务执行。

  4. 重复上述步骤： 重复执行上述步骤，直至任务队列为空。

深浅拷贝 ​

  • 浅拷贝指的是将一个对象的属性值复制到另一个对象，如果有的属性的值为引用类型的话，那么会将这个引用的地址复制给对象，因此两个对象会有同一个引用类型的引用。浅拷贝可以使用 Object.assign 和展开运算符来实现。

  • 深拷贝相对浅拷贝而言，如果遇到属性值为引用类型的时候，它新建一个引用类型并将对应的值复制给它，因此对象获得的一个新的引用类型而不是一个原有类型的引用。深拷贝对于一些对象可以使用 JSON 的两个函数来实现，但是由于 JSON 的对象格式比 JS 的对象格式更加严格，所以如果属性值里边出现函数或者 Symbol 类型的值时，会转换失败。

函数柯里化 ​

函数柯里化（Currying）是一种将接受多个参数的函数转化为一系列接受单个参数的函数的技术。通过柯里化，可以创建一个接受一个参数并返回一个新函数的过程，新函数接受下一个参数，依此类推，直到所有参数都被收集完成，最终返回结果。

函数柯里化通常通过递归或闭包实现。它可以带来一些优势，包括代码的简化、参数的延迟传递以及创建更灵活的函数。

以下是一个简单的 JavaScript 函数柯里化的示例：

javascript
// 非柯里化版本
function add(a, b, c) {
  return a + b + c;
}

console.log(add(1, 2, 3)); // 输出：6

// 柯里化版本
function curryAdd(a) {
  return function(b) {
    return function(c) {
      return a + b + c;
    };
  };
}

console.log(curryAdd(1)(2)(3)); // 输出：6

递归实现：

function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    } else {
      return function(...moreArgs) {
        return curried.apply(this, args.concat(moreArgs));
      };
    }
  };
}

JavaScript 中倒计时的纠偏实现 ​

一般通过 setTimeout 和 setInterval 方法来实现一个倒计时效果。但是使用这些方法会存在时间偏差的问题，这是由于 JavaScript 的程序执行机制造成的，setTimeout 和 setInterval 的作用是隔一段时间将回调事件加入到事件队列中，事件并不是立即执行的，它会等到当前执行栈为空的时候再取出事件执行，因此事件等待执行的时间就是造成误差的原因。

一般解决倒计时中的误差的有这样两种办法：

  • 第一种是通过前端定时向服务器发送请求获取最新的时间差，以此来校准倒计时时间。

  • 第二种方法是前端根据偏差时间来自动调整间隔时间的方式来实现的。这一种方式首先是以 setTimeout 递归的方式来实现倒计时，然后通过一个变量来记录已经倒计时的秒数。每一次函数调用的时候，首先将变量加一，然后根据这个变量和每次的间隔时间，我们就可以计算出此时无偏差时应该显示的时间。然后将当前的真实时间与这个时间相减，这样我们就可以得到时间的偏差大小，因此我们在设置下一个定时器的间隔大小的时候，我们就从间隔时间中减去这个偏差大小，以此来实现由于程序执行所造成的时间误差的纠正。
